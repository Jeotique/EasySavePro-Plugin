EasySavePro Plugin

EasySavePro is a robust, asynchronous save/load system for Unreal Engine 5. Designed for flexibility and performance, it allows you to save any actor, handle open worlds, manage persistent player data, and capture save thumbnails without freezing your game.

2. Getting Started (3 Steps)

Follow this quick start guide to get your saving system running in less than 5 minutes.

01

Step 1: Enable the Plugin

Go to Edit > Plugins and enable EasySavePro. Restart the editor.

Crucial Requirement Most nodes require the EasySaveSubsystem context. To get it, right-click in your Blueprint (Context Sensitive checked) and search for "Get EasySaveSubsystem". Connect this return value to the "Target" pin of any EasySave node.
02

Step 2: Mark Actors to Save

To save an actor (Player, Chest, Enemy), simply add the EasySaveComponent to it in Blueprint.

  • The component automatically adds the tag "SaveMe" to the actor.
  • It handles saving Transform, Physics, and variables marked as "SaveGame".
03

Step 3: Implement Interface (Optional)

To run custom logic (e.g., refill health, reset AI behavior), add the EasySaveInterface to your Actor Blueprint.

OnSaveStarted Called before saving. Use to update variables.
OnActorLoaded Called after this specific actor is restored.
OnGameLoaded Called after the entire world is finished loading.

6. Best Practices

  1. Handle Level Travel Properly Use the OpenLevelWithData node provided by the plugin instead of the standard Unreal node if you want to keep inventory/health across maps (e.g. going from Hub to Dungeon).
  2. Unique IDs & Duplication The EasySaveComponent generates a GUID automatically. However, if you duplicate an actor in the level editor (Alt+Drag), click "Force Generate ID" in the details panel to ensure no ID conflicts occur.
  3. Variable Selection Only variables marked with the "SaveGame" checkbox (Advanced Display in BP details) are saved! If a variable isn't saving, check this first.
